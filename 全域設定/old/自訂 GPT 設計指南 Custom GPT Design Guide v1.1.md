# Custom GPT Design Guide v1.1

## 設計目的

本文件並非用於定義某一個 Custom GPT 的規格，而是整理設計 Custom GPT
的共同理念、原則與方法。

其目的為建立一致的設計思維，使不同領域的 Custom GPT
能維持相同的架構、清楚的職責與良好的可維護性。

本文件不參與 Runtime 執行，而是作為開發階段的設計指南。

------------------------------------------------------------------------

# 核心理念

## 1. 先定義職責，再定義能力

不要從「希望 GPT 會做什麼」開始設計，而應先回答：

-   它是什麼？
-   它解決什麼問題？
-   它負責什麼？
-   它不負責什麼？

職責明確後，再決定需要哪些工作流程與知識。

------------------------------------------------------------------------

## 2. Context（情境）比能力更重要

設計的重點不是增加 AI 能力，而是提供完成任務所需的 Context（情境）。

好的設計，是讓 AI 在正確的情境下使用正確的能力。

------------------------------------------------------------------------

## 3. 分離不同層次的職責

每一層只回答自己的問題，不跨越其他層。

能引用，不重複定義（DRY）。

------------------------------------------------------------------------

## 4. 保持最小必要結構

每一層保持單一職責（Single Responsibility）。

新增抽象之前，先確認：

-   解決了什麼問題？
-   現有架構是否已能解決？
-   是否真的需要新增一層？

若無法回答，不應新增。

------------------------------------------------------------------------

## 5. 問題驅動，而非想法驅動

優先依據真實使用經驗持續改善。

修改應建立於：

-   真實需求
-   使用經驗
-   問題
-   邊界不清
-   維護成本

而不是因為想到新的設計。

------------------------------------------------------------------------

# 設計層（Design Layers）

Custom GPT 應依照不同層次分離職責。

每一層回答不同的問題，保持自己的責任範圍，能引用，不重複定義。

## 1. 個人化指令（Personal Instruction）

**回答的問題**

AI 共同遵循哪些原則？

**主要職責**

-   共通語言
-   回答方式
-   分析原則
-   共同行為

**不包含**

-   特定領域知識
-   特定 GPT 定位
-   個別工作流程

------------------------------------------------------------------------

## 2. 自訂 GPT（Custom GPT）

**回答的問題**

它是什麼？負責什麼？

**主要職責**

-   定位（Purpose）
-   職責（Responsibility）
-   適用範圍
-   邊界

**不包含**

-   詳細知識
-   工作流程細節

------------------------------------------------------------------------

## 3. 工作流程（Workflow）

**回答的問題**

它如何完成工作？

**主要職責**

-   工作流程
-   任務順序
-   決策流程
-   例外處理

**不包含**

-   GPT 定位
-   領域知識

------------------------------------------------------------------------

## 4. 知識（Knowledge）

**回答的問題**

它需要知道什麼？

**主要職責**

-   領域知識
-   專業資料
-   規範
-   世界觀
-   參考資料

**不包含**

-   工作流程
-   共同行為
-   GPT 定位

------------------------------------------------------------------------

## 5. 工具（Tool）

**回答的問題**

它目前可以取得或操作什麼？

**主要職責**

-   外部工具
-   即時資料
-   系統能力
-   外部資源

**不包含**

-   工作流程
-   知識管理
-   GPT 定位

------------------------------------------------------------------------

## 設計層的關係

上層定義原則，下層提供能力。

每一層可引用所需的下層內容，但不應重新定義其內容。

例如：

-   個人化指令可影響所有層。
-   自訂 GPT 可引用工作流程與知識。
-   工作流程可引用知識。
-   知識不應重新定義工作流程。
-   工具提供能力，由上層決定是否使用。

------------------------------------------------------------------------

# 設計流程

建立新的 Custom GPT 時，建議依照以下順序思考：

1.  定義定位（Purpose）
2.  定義職責（Responsibility）
3.  設計工作流程（Workflow）
4.  整理知識（Knowledge）
5.  最後才撰寫 Prompt、規則或知識庫

不要從 Prompt 開始設計。

------------------------------------------------------------------------

# 設計原則

設計過程應優先回答：

-   它是什麼？
-   它解決什麼問題？
-   它如何工作？
-   它需要知道什麼？

而不是：

-   要寫哪些 Prompt？
-   要加入哪些技巧？

------------------------------------------------------------------------

# 最後

不要直接設計 GPT。

而是先定義它的職責，再決定它需要哪些工作流程與知識。

真正成熟的 Custom
GPT，不是功能最多，而是每一層都有清楚的職責，每一項設計都有存在的理由。
