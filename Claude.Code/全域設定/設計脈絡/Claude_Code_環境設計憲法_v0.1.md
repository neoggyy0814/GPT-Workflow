# Claude Code 環境設計憲法 v0.1

本文件是《通用_AI_控制設計指南 v0.5》在 Claude Code 平台(CLAUDE.md / Skill / Hook)上的具體應用,不重新定義 v0.5 已有的哲學與原則,只處理「這些原則在這個平台上該怎麼落地」。若本文件與 v0.5 內容有出入,以 v0.5 為準,本文件應被視為 v0.5 的下位應用,不是平行的另一套標準。

本文件分兩部分,對應 v0.5 自己的區分方式(拿掉這一條,模型行為會不會改變):

- **第一部分:分類決策程序**——給你用的設計時工具,不載入環境本身。
- **第二部分:不變核心原則**——會被載入的內容,放在你的全域或根目錄 CLAUDE.md 最上方。

# 一、分類決策程序(設計時工具,不載入)

任何一段你想寫進這個環境的內容,依序回答以下問題,決定它該去哪:

### 第一問:這件事必須每次都被做到、不能靠模型自己決定嗎?

→ 是:寫成 **Hook**。(對應 v0.5 設計原則第一級:機制限制與撰寫問題的區分——這類需求本質是「已知無法單靠指令解決」,直接交給不經模型判斷的外部程式。)

→ 否:進第二問。

### 第二問:這是每個 session 都該知道、且無法從程式碼/設定檔輕易推得的穩定事實嗎?(建置指令、慣例、專案結構、「一律做 X」規則)

→ 是,且不是多步驟流程:寫進 **CLAUDE.md 本體**。

→ 是多步驟流程,或只對某個子目錄/情境有意義:進第三問。

### 第三問:是多步驟流程,或路徑限定的規則嗎?

→ 是:寫成 **Skill**(多步驟流程)或 `.claude/rules/` 底下的路徑限定檔(只對特定子目錄有意義的規則)。

### 第四問(每一問都要附帶檢查):這是否已經在別處定義過?(程式碼本身、README、其他層級的 CLAUDE.md)

→ 是:改為引用,不複寫。(對應設計原則第四級:單一事實來源)

→ 若這份內容是「衍生自」另一份權威來源(例如你摘要一份更長的規格文件):明文加註「與權威來源牴觸時,以權威來源為準」。(對應設計原則第五級:從屬聲明)

### 第五問(新增任何檔案/層級前都要問):現有結構真的不夠用嗎?

→ 答不出具體理由:不新增。(對應設計原則第二級:最小必要結構)

### 第六問(修改既有內容前都要問):這解決了哪個實際發生過的問題?

→ 答不出來:不動手。(對應設計哲學:問題驅動,而非想法驅動)

# 二、不變核心原則(會被載入,放在全域/根目錄 CLAUDE.md 最上方)

以下內容是本環境的不可覆蓋核心。任何專案層 CLAUDE.md、路徑限定規則、Skill、或 auto memory,若與本節內容衝突,一律以本節為準。

## 1. 優先權聲明

本節內容為本環境最高優先權。下層(專案層 CLAUDE.md、`.claude/rules/`、auto memory)僅能補充,不能與本節牴觸。若發現牴觸,視為下層內容有誤,應修正下層,而非視為本節失效。

**此聲明存在的理由**:多個 CLAUDE.md 層級之間,若無明文優先權聲明,系統可能依「後載入者優先」處理衝突,而不是自動以更上層或更早定義的內容為準——這代表若不寫這一條,你不能預設自己的核心規則天然佔優勢。

## 2. Auto memory 從屬聲明

Auto memory(模型於 session 中自行寫下的筆記)從屬於本節與人工撰寫的 CLAUDE.md。若 auto memory 內容與本節或人工撰寫規則衝突,以人工撰寫規則為準;發現衝突時,應清除或修正該筆記,而非任其與人工規則並存。

**此聲明存在的理由**:auto memory 是模型自己生成的內容,產生時間在你設計 CLAUDE.md 之後、你未必即時看到的地方持續發生,若不明文從屬,它的分歧風險與人工撰寫的衍生檔案不同——來源是模型自身,不是人為疏失。

## 3. 職責邊界聲明

每一份 CLAUDE.md、Skill、Hook 只回答自己該回答的問題:CLAUDE.md 本體只放無條件常駐的穩定事實;Skill 只放多步驟流程;Hook 只放不容模型自主決定的強制動作。新增任何內容前,先確認是否已有涵蓋既有定義,不重複描述。

## 4. 修改紀律聲明

任何對本環境(CLAUDE.md、Skill、Hook)的修改,應能回答「這解決了什麼實際發生過的問題」。無法回答者,不執行修改。

## 待辦

- 本文件第一部分(分類決策程序)是設計時工具,實際寫 CLAUDE.md/Skill/Hook 時,建議把它當檢查清單使用,不需要也不應該把它本身塞進環境裡。
- 第二部分(不變核心原則)目前是通用版本,尚未依你實際專案內容(學校 IT 事務、Scratch/Python 教學等)填入具體規則,那些屬於下一步「整理別人的 CLAUDE.md 抽象化」之後,你自己環境的專案層內容,不屬於這份「憲法」層級。
