用 Claude Code 的人越來越多，但很多人只會叫它「幫我寫功能」。其實Anthropic官方整理了 52 組提示詞，我全部濃縮翻譯成中文版、讓大家參考🥰 （【】裡換成你自己的檔案或功能名稱就行）
【1/3】剛接手新專案 1. 摸清全貌 —「給我這個專案的總覽：架構、主要資料夾、各部分怎麼串」 2. 刪東西前先問 —「如果我把【要刪的功能】刪掉，會有什麼地方跟著壞？」 3. 白話問功能 —「使用者按下【某個按鈕】後，從畫面到結果發生了什麼事？」 4. 先規劃再動手 —「規劃怎麼改造【模組】來【目標】，先列出要改的檔案、但別動手」 5. 列出例外狀況 —「列出【某流程】會遇到的錯誤、空白、極端情況，給設計參考」
【2/3】寫功能的時候，讓它抄你的現有功能
6. 照現有寫法做 —「看【現有功能】怎麼寫的，照同樣模式做一個【新功能】」 7. 做內部小工具 —「用網頁技術做一個【小工具】，做完打開給我看」 8. 寫測試修錯 —「幫【某個檔案】寫測試、跑一遍、把失敗的修掉」 9. 照數字優化 —「優化【某功能】，把反應時間從【現況】降到【目標】以下」 10. 送出前自己檢查 —「檢查我還沒送出的修改，把看起來有風險的地方標出來」 11. 資安檢查 —「派一個分身專門檢查【某資料夾】的安全問題，回報發現」 12. 讓它記住規矩 —「你一直犯【某個錯】，把這條規矩寫進設定檔，之後別再犯」
【3/3】出包了，都可以叫它幫忙 13. 自動寫提交說明 —「把這些修改提交，說明幫我總結我做了什麼」 14. 線上出事調查 —「【某功能】一小時前開始出錯，查紀錄、最近的更新、設定改動，說最可能的原因」 15. 截圖問診 —「這是【某後台】的截圖，帶我看為什麼【某個東西】掛了、給我修復步驟」 16. 分析資料檔 —「讀這個【資料檔】，總結重點，做成有圖表的網頁打開」 17. 接外部工具 —「幫我接上【某工具】，讓你能直接讀我的【某資料】」 18. 收尾沉澱 —「總結我們這次做了什麼，建議哪些可以加進設定檔」
